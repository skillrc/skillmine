## рҹҺЁ Usage Guidelines

### Terminal Compatibility
- All ASCII art uses standard Unicode box-drawing characters
- Compatible with UTF-8 terminals (most modern terminals)
- Fallback: Use emoji-free versions for legacy systems

### Color Usage in CLI
```bash
# Primary brand color (orange) for:
- Command names
- Success messages
- Skill names

# Cyan for:
- URLs and links
- Interactive prompts
- Highlighted values

# Gray for:
- Secondary information
- Timestamps
- File paths

# Semantic colors for:
- Green: Success, checkmarks, completion
- Yellow: Warnings, pending states
- Red: Errors, conflicts
- Blue: Info messages, tips
```

### Logo Placement
- **README header**: Use full banner
- **CLI output**: Use compact logo with command
- **Documentation**: Use inline version `вӣҸ skillmine`
- **Error messages**: No logo, just text

### Typography
- **Monospace fonts**: Required for ASCII art alignment
- **Recommended fonts**: 
  - JetBrains Mono
  - Fira Code
  - Cascadia Code
  - SF Mono
  - Consolas

## рҹ“җ Design Rationale

### Why Mining Theme?
- **Metaphor**: "Mining" for skills, extracting value
- **Pickaxe**: Universal symbol of digging/extraction
- **Ore/Gems**: Skills are valuable resources to collect
- **Rust connection**: Orange color ties to Rust language

### Why These Colors?
- **Orange (#FF6B35)**: Energetic, stands out in terminal, Rust heritage
- **Cyan (#00D9FF)**: High contrast on dark backgrounds, tech-forward
- **Dark slate**: Professional, easy on eyes for long terminal sessions

### Design Principles
1. **Terminal-first**: Everything must work in monospace
2. **High contrast**: Readable in both light and dark themes
3. **Minimal**: No clutter, clean lines
4. **Distinctive**: Not another purple gradient SaaS brand
5. **Developer-friendly**: Feels like a tool, not a toy
