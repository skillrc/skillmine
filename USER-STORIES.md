# SPM 用户故事文档

**项目**: Skill Package Manager (SPM)  
**版本**: 1.0  
**日期**: 2026-03-12  

---

## 用户故事清单

### 史诗 1：配置管理 (Epic: Configuration Management)

作为 AI 编程助手用户，我希望通过声明式配置文件管理技能，以便版本控制和团队协作。

#### US-001: 初始化配置
**作为** 新用户  
**我希望** 运行 `spm init` 创建默认配置文件  
**以便** 快速开始管理技能  

**验收标准**:
- [ ] 命令创建 `~/.config/spm/skills.yaml`
- [ ] 文件包含示例配置和注释
- [ ] 如果文件已存在，提示是否覆盖
- [ ] 支持 `--global` (默认) 和 `--local` (项目级) 选项

**优先级**: P0  
**估算**: 2 点  

---

#### US-002: 查看当前配置
**作为** 用户  
**我希望** 运行 `spm config show` 查看当前配置  
**以便** 了解已安装的技能列表  

**验收标准**:
- [ ] 显示配置文件路径
- [ ] 以表格形式列出所有技能
- [ ] 显示每个技能的来源和版本
- [ ] 支持 `--json` 输出格式

**优先级**: P1  
**估算**: 1 点  

---

#### US-003: 添加技能到配置
**作为** 用户  
**我希望** 运行 `spm add <repo>` 添加技能  
**以便** 声明式地添加新技能  

**验收标准**:
- [ ] 支持 `user/repo` 格式
- [ ] 自动解析最新版本
- [ ] 更新 `skills.yaml` 文件
- [ ] 支持 `--tag`, `--branch`, `--commit` 选项
- [ ] 验证仓库可访问性

**优先级**: P0  
**估算**: 3 点  

---

#### US-004: 从配置中移除技能
**作为** 用户  
**我希望** 运行 `spm remove <name>` 移除技能  
**以便** 清理不再需要的技能  

**验收标准**:
- [ ] 从 `skills.yaml` 中移除条目
- [ ] 提示是否删除已安装的本地文件
- [ ] 如果技能不存在，给出友好提示
- [ ] 支持 `--yes` 跳过确认

**优先级**: P1  
**估算**: 2 点  

---

### 史诗 2：安装管理 (Epic: Installation Management)

作为用户，我希望并发安装多个技能，以便快速设置开发环境。

#### US-101: 安装所有声明的技能
**作为** 用户  
**我希望** 运行 `spm install` 安装所有技能  
**以便** 一次性设置完整环境  

**验收标准**:
- [ ] 读取 `skills.yaml` 配置
- [ ] 并发下载（默认 5 个并发）
- [ ] 显示实时进度条
- [ ] 下载到 `~/.config/spm/cache/`
- [ ] 验证下载完整性
- [ ] 失败时显示详细错误信息
- [ ] 支持 `--force` 强制重新安装
- [ ] 支持 `--dry-run` 预览安装

**优先级**: P0  
**估算**: 8 点  

---

#### US-102: 安装指定技能
**作为** 用户  
**我希望** 运行 `spm install <name>` 安装特定技能  
**以便** 单独安装或更新某个技能  

**验收标准**:
- [ ] 只安装指定的技能及其依赖
- [ ] 如果技能未在配置中，提示添加
- [ ] 支持通配符匹配多个技能

**优先级**: P1  
**估算**: 3 点  

---

#### US-103: 查看安装状态
**作为** 用户  
**我希望** 运行 `spm list` 查看已安装的技能  
**以便** 了解当前环境状态  

**验收标准**:
- [ ] 显示技能名称、版本、安装时间
- [ ] 区分已安装和未安装的技能
- [ ] 显示是否有可用更新
- [ ] 支持 `--json` 和 `--tree` 格式

**优先级**: P1  
**估算**: 2 点  

---

#### US-104: 清理缓存
**作为** 用户  
**我希望** 运行 `spm clean` 清理缓存  
**以便** 释放磁盘空间  

**验收标准**:
- [ ] 删除未使用的缓存文件
- [ ] 显示释放的空间大小
- [ ] 支持 `--all` 清理所有缓存
- [ ] 支持 `--dry-run` 预览清理

**优先级**: P2  
**估算**: 2 点

### 史诗 3：版本控制 (Epic: Version Control)

作为团队成员，我希望锁定技能版本，以便确保协作环境一致。

#### US-201: 生成锁定文件
**作为** 用户  
**我希望** 运行 `spm freeze` 生成锁定文件  
**以便** 记录当前精确版本  

**验收标准**:
- [ ] 生成 `skills.lock` 文件
- [ ] 记录每个技能的 Git commit SHA
- [ ] 记录锁定时间戳
- [ ] 支持 `--output` 指定输出路径
- [ ] 锁定文件格式清晰可读

**优先级**: P0  
**估算**: 3 点  

---

#### US-202: 从锁定文件恢复
**作为** 用户  
**我希望** 运行 `spm thaw` 从锁定文件恢复  
**以便** 复现特定环境  

**验收标准**:
- [ ] 读取 `skills.lock` 文件
- [ ] 检出指定的精确版本
- [ ] 忽略 `skills.yaml` 中的版本声明
- [ ] 支持 `--input` 指定锁定文件路径

**优先级**: P0  
**估算**: 3 点  

---

#### US-203: 检查更新
**作为** 用户  
**我希望** 运行 `spm outdated` 查看可更新的技能  
**以便** 了解最新版本情况  

**验收标准**:
- [ ] 对比当前版本和远程最新版本
- [ ] 显示版本差异（当前 → 最新）
- [ ] 支持 `--json` 输出
- [ ] 支持 `--all` 显示所有技能（包括最新）

**优先级**: P1  
**估算**: 3 点  

---

#### US-204: 更新技能
**作为** 用户  
**我希望** 运行 `spm update` 更新技能  
**以便** 获取最新功能和修复  

**验收标准**:
- [ ] 更新所有技能到最新版本
- [ ] 支持 `spm update <name>` 更新特定技能
- [ ] 显示更新前后的版本对比
- [ ] 支持 `--dry-run` 预览更新
- [ ] 失败时保留原版本

**优先级**: P1  
**估算**: 5 点

### 史诗 4：多目标同步 (Epic: Multi-Target Sync)

作为用户，我希望将技能同步到不同的 AI 编程助手，以便在不同工具中使用相同配置。

#### US-301: 同步到 Claude Code
**作为** Claude Code 用户  
**我希望** 运行 `spm sync --target=claude` 同步技能  
**以便** 在 Claude Code 中使用管理的技能  

**验收标准**:
- [ ] 创建 `~/.claude/skills/` 目录结构
- [ ] 使用符号链接指向缓存目录
- [ ] 验证 Claude Code 技能格式
- [ ] 支持 `--force` 强制重新同步
- [ ] 显示同步的技能数量

**优先级**: P0  
**估算**: 5 点  

---

#### US-302: 同步到 OpenCode
**作为** OpenCode 用户  
**我希望** 运行 `spm sync --target=opencode` 同步技能  
**以便** 在 OpenCode 中使用管理的技能  

**验收标准**:
- [ ] 创建 `~/.config/opencode/skills/` 目录结构
- [ ] 使用符号链接指向缓存目录
- [ ] 验证 OpenCode 技能格式
- [ ] 支持 `--force` 强制重新同步

**优先级**: P0  
**估算**: 3 点  

---

#### US-303: 同步到自定义路径

**我希望** 运行 `spm sync --target=custom --path=/path` 同步到自定义位置  
**以便** 支持非标准安装路径  

**验收标准**:
- [ ] 支持任意目标路径
- [ ] 自动创建目标目录
- [ ] 验证目标路径可写

**优先级**: P2  
**估算**: 2 点

#### US-304: 自动同步
**作为** 用户  
**我希望** 在 `spm install` 后自动同步到目标平台  
**以便** 减少手动操作  

**验收标准**:
